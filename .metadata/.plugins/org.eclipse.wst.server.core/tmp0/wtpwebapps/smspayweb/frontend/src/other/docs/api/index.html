TestSMS client API v1

Scroll down for code samples, example requests and responses. Select a language for code samples from the tabs above or the mobile navigation menu.

The Test SMS API allows you to integrate directly with your SMS system and use it in your UI. It is organized around HTTP REST API. Our API has predictable resource-oriented URLs, accepts JSON-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

Introduction

First thing is to create your free Test SMS account, its free for testing and integration process. In order to activate your account, we would need further information regarding your business and how our Test SMS Platform will be of help to your services.

Once your account is activated you are ready to go live and using Test SMS API.

Base URLs:

  • https://my.testsms.com/api

Terms of service Email: TestSMS Support Web: TestSMS Support License: Apache 2.0

Authorization

Test SMS uses OAuth2 over https to manage authentication and authorisation.OAuth2 is a protocol that lets external applications request permission from another Test SMS user to send requests on their behalf without getting their password.

New to OAuth2? See more at official OAuth2 site and using barrer token.
All requests to the TestSMS API require a access_token for authentication. You need your own credentials to generate access token. This token expires in 24 hours and you need to request new one when token expired.

POST /v1/token

Post email and password to get Access Token. Now that you have an access_token you can interact with your Test SMS account via the API.
To do so, you must simply append the access token to the header of any API request:
Authorization: Bearer {access_token}

Body parameter

{
  "email": "name@comapny.com",
  "password": "******"
}

» password body string true Your password for TestSMS portal

GET /v1/mccmnc

Use to get available networks for test and all test scenarios including MNP (ported numbers). From returned list, most important attributes are mccmnc and mccmncOriginal if exists. Param mccmnc is required for creating test for specific country and network operator. Param mccmncOriginal, if exists, represents that ported number is available for testing.

Note that refreshing period of available operator networks for testing is few minutes.

POST /v1/createTest

Post networks, array of mccmnc to get phone numbers for test. Set callback URL to get test results.
Create test cases for list of networks. Be sure that sent networks are from recent call to MccMnc API, so all requested networks have online numbers. Each created test contains id, messageId, msisdn. To test your route send text that contains messageId to msisdn (phone number). Get results of each test using id in smsTest/{id} API call.
Set callbackUrl as your endpoint, so our service can send you immediate response of tested cases.

Body parameter

{
  "callbackUrl": "string",
  "networks": [
    {
      "mccmnc": "string",
      "mccmncOriginal": "string"
    }
  ]
}

Parameters

Name In Type Required Description
body body CreateTestsRequest true Post networks, array of mccmnc, mccmncOriginal, callbackUrl to get phone numbers for tests
» callbackUrl body string false none
» networks body [CreateTest] false none
»» mccmnc body string false none
»» mccmncOriginal body string false none

GET /v1/smsTest/{id}

Get result to check route quality and network availability. Param receiptStatus used to explain how SMS arrived on device. Values for ** receiptStatus** are WAIT(0), POSITIVE(1), NEGATIVE(5). Note that our system is waiting for sms up to 2 hours and than automatically set NEGATIVE status.
